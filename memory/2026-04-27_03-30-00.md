# 2026-04-27 个人博客项目搭建

## 1. 本次任务大纲

从零搭建一个基于 Next.js 15 的极简现代风个人博客网站，要求界面精美、后端稳定、安全性高。核心功能包括：MDX 文章系统、标签分类、Cmd+K 全文搜索、暗色模式、SEO 优化（Sitemap/RSS/OpenGraph）。

## 2. 如何解决

### 技术栈选型
| 技术 | 用途 |
|------|------|
| Next.js 16 (App Router, webpack mode) | 框架 |
| Velite + MDX | 内容管理（Zod 类型安全） |
| Tailwind CSS v4 + CSS Variables | 样式 + 主题 |
| Framer Motion | 动画 |
| FlexSearch 概念 / 客户端搜索 | 搜索（Cmd+K 弹窗） |
| next-themes | 暗色模式 |
| Rehype Pretty Code (Shiki) | 代码高亮 |
| Lucide Icons | 图标 |

### 实施步骤
1. `npx create-next-app@latest` 初始化 + 安装 ~15 个依赖
2. 配置 `velite.config.ts`（Post schema + MDX rehype 插件链）
3. 配置 `next.config.ts`（Velite webpack 插件 + 安全响应头）
4. 创建全局布局：Geist 字体 + ThemeProvider + Header（毛玻璃导航栏）+ Footer
5. 创建 MDX 组件映射：Callout 提示框、代码块增强、Image 组件
6. 开发 6 个页面：首页、文章列表、文章详情（+侧边 TOC）、标签云、标签筛选、关于
7. 实现 Cmd+K 搜索弹窗（键盘导航 + 动画）
8. SEO：sitemap.ts、robots.ts、feed.xml（RSS）
9. 安全加固：CSP、X-Frame-Options、X-Content-Type-Options、Referrer-Policy

### 遇到的问题和解决
- **Next.js 16 默认 Turbopack**：Velite 需要 webpack 插件，通过 `--webpack` 标志和 `turbopack: {}` 配置解决
- **lucide-react 移除品牌图标**：新版不再导出 `Github` 图标，改用内联 SVG 替代

### 构建结果
- 所有 21 个路由成功静态生成
- 3 篇示例文章（含代码高亮、Callout 组件）
- Velite 构建 ~490ms

## 3. 方案能否被复用

### 高度可复用的场景
- **任何个人技术博客**：替换 `content/posts/` 下的 MDX 文件即可
- **文档网站**：调整 Velite schema 添加 sidebar 字段
- **作品集网站**：添加 projects collection

### 复用注意事项
- `lib/constants.ts` 需修改站点名称、URL、作者信息
- Next.js 16+ 需要 `--webpack` 标志配合 Velite 使用
- `velite.config.ts` 的 rehype 插件链可按需增减
- 部署到 Vercel 时需确保构建命令包含 `--webpack`
- 暗色模式基于 CSS 变量，可在 `globals.css` 中自定义色彩
