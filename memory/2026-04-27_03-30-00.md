# 2026-04-27 个人博客项目 — 完整记录

## 1. 本次任务大纲

从零搭建一个基于 Next.js 的极简现代风个人博客网站，完成从开发到上线的全流程：
- 界面精美（极简现代风、暗色模式、Framer Motion 动画）
- 后端稳定（纯静态导出，无服务端）
- 安全性高（CSP、安全响应头、无数据库/无用户输入）
- 部署到 Cloudflare Pages，国内可访问，免备案
- 提供 `npm run new / delete / publish` 一键管理文章

## 2. 如何解决

### 技术栈
| 技术 | 用途 |
|------|------|
| Next.js 16 (App Router, `--webpack` mode) | 框架 |
| Velite + MDX | 内容管理（Zod 类型安全） |
| Tailwind CSS v4 + CSS Variables | 样式 + 主题系统 |
| Framer Motion | 页面/卡片动画 |
| 客户端搜索（标题+描述+标签匹配） | Cmd+K 搜索弹窗 |
| next-themes | 暗色模式（系统自适应 + 手动切换） |
| Rehype Pretty Code (Shiki) | 代码块语法高亮 |
| Cloudflare Pages | 部署（静态导出 `output: "export"`） |

### 项目结构
```
blog/
├── app/                    # 页面（首页、文章列表/详情、标签、关于）
├── components/             # 组件（header、footer、theme-toggle、post-card、toc、search-dialog、callout、mdx-content）
├── content/posts/          # MDX 文章（写文章放这里）
├── lib/                    # 工具函数 + 站点配置
├── scripts/                # 管理脚本（new-post、delete-post、publish、generate-rss）
├── public/                 # 静态资源 + _headers 安全头 + robots.txt
├── velite.config.ts        # Velite 内容 schema
├── next.config.ts          # Next.js 配置（静态导出 + webpack Velite 插件）
└── out/                    # 构建产物（Cloudflare 部署此目录）
```

### 关键配置文件说明
- **`velite.config.ts`**: Post schema（title, slug, date, description, tags, published, body），MDX rehype 插件链（slug → pretty-code → autolink-headings）
- **`next.config.ts`**: `output: "export"` 静态导出 + `images: { unoptimized: true }` + Velite webpack 插件 + `turbopack: {}`
- **`lib/constants.ts`**: 站点名称、URL、作者信息等集中配置
- **`public/_headers`**: Cloudflare Pages 安全响应头（CSP、X-Frame-Options 等）
- **`scripts/generate-rss.js`**: 构建后生成 `feed.xml` + `sitemap.xml` 到 `out/`

### 遇到的问题和解决方案
1. **Next.js 16 默认 Turbopack**: Velite 需要 webpack 插件 → 配置 `turbopack: {}` + `--webpack` 标志
2. **lucide-react 移除品牌图标**: `Github` 不再导出 → 改用内联 SVG
3. **静态导出不支持 `robots.ts`/`sitemap.ts`**: 动态路由函数报错 → 改为 `public/robots.txt` 静态文件 + `scripts/generate-rss.js` 构建时生成
4. **静态导出不支持 Route Handler**: `feed.xml/route.ts` 不兼容 → 改为构建后脚本生成
5. **Cloudflare 创建项目选错入口**: Workers 模式会用 `opennextjs-cloudflare` 导致失败 → 必须从 Pages 入口创建，Framework preset 选 `None`
6. **SSH 推送 GitHub**: HTTPS 需交互式认证不方便 → 添加 SSH public key 到 GitHub

### 文章管理命令
```bash

```

## 3. 方案能否被复用

### 高度可复用的场景
- **任何个人技术博客**: 修改 `lib/constants.ts` + 替换 `content/posts/` 即可
- **文档网站**: 调整 Velite schema 添加 sidebar/category 字段
- **作品集网站**: 添加 projects collection
- **团队博客**: 添加 author 字段到 schema

### 复用清单
1. 修改 `lib/constants.ts`（站点名称、URL、GitHub、邮箱）
2. 修改 `scripts/generate-rss.js` 顶部的 SITE_URL / SITE_NAME
3. 修改 `public/robots.txt` 中的 sitemap URL
4. 替换 `content/posts/` 下的示例文章
5. 替换 `public/favicon.ico`
6. Cloudflare Pages 创建时**必须选 Pages 入口**，preset 选 `None`
7. Next.js 16+ 配合 Velite 必须用 `--webpack` 标志
8. `globals.css` 中的 CSS 变量可自定义色彩体系
